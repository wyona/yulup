Answers

Troubleshooting Installation

Q. When I click on one of the download links below, Firefox tells me something along the lines of "Software installation is currently disabled. Click Edit Options... to enable it and try again." When clicking on "Edit Options...", the preferences dialog is opened but there is no relevant option to reenable software installation.

A. The option to enable software installation was removed from the user interface, without implementing a replacement. This is a known bug and is fixed in Firefox 2 alpha 2. For users of Firefox 1.5.0.x, a workaround is to open the URI "about:config", scroll down to the option "xpinstall.enabled", right click on that name, and select "Toggle" from the context menu, such that the "Value" column reads "true".

Note that this bug must not necessarily show up with your installation, since the "xpinstall.enabled" is set to "true" by default, and could only have been changed by you yourself (or your administrator) in an earlier release or manually by using "about:config".

Q. Cannot upgrade to a newer version although a newer version is indeed available (Extension Manager reports: "No updates were found for Wyona Yulup").

A. Clear your browser cache (Preferences -> Privacy -> Cache -> Clear Cache Now) then try again to update. Alternative 1: Uninstall Yulup and reinstall afresh from the Yulup website. Alternative 2: Delete your local profile and restart Firefox (warning: this deletes all your bookmarks, etc., therefore you might want to create a backup copy of your profile folder first, and afterwards migrate your old bookmarks to your new profile).

Q. How to upgrade from an older to a newer version.

A. Click on the Firefox/Mozilla Tools menu, then on the Extensions menu item, and then click "Find Updates".

Q. Firefox could not install the file at http:// because: Not a valid install package.

A. This seems to be a Firefox bug. After 4 or 5 times re-trying to install it usually suddenly works.

Running Yulup

Q. Yulup does not detect Neutron enabled pages or network connection fails.

A. If you have the FireBug extension installed next to Yulup, please disable FireBug or uninstall it. FireBug is known to break Yulup in its current release. We are currently implementing a specific Yulup prefix as a workaround. Please stay tuned.

Q. Although the cursor is placed inside the source editor, no text can be entered.

A. This is bug 5153 and is fixed in Yulup version 0.2.2. You might want to consider updating.

In case you can't update right now, simply position the cursor inside the source view and hit TAB once.
